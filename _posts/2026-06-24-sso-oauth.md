## 한눈에 비교


| 용어    | 의미                  | 핵심 키워드 |
| ----- | ------------------- | ------ |
| SSO   | 한 번 로그인으로 여러 시스템 이용 | 통합 로그인 |
| OAuth | 다른 서비스에 로그인 권한 위임   | 권한 위임  |


---


## SSO (Single Sign-On)


한 번 로그인하면 연결된 여러 시스템을 다시 로그인하지 않고 이용할 수 있는 인증 방식이다.

- 한 번 로그인 → 여러 시스템 사용
- 예) 회사 그룹웨어에 로그인하면 → 메일, 인사시스템, 전자결재까지 별도 로그인 없이 사용
- 핵심: **통합 로그인**

---


## OAuth


내 로그인 정보(비밀번호)를 직접 넘기지 않고, **인증 권한만 위임**하는 방식이다.

- 예) "구글로 로그인", "카카오로 로그인" 버튼
- 비밀번호를 제3의 서비스에 넘기지 않는다
- 구글/카카오가 "이 사람 맞아요"라고 확인해주고, 그 확인 결과(토큰)를 넘겨준다
- 핵심: **권한 위임**

```plain text
사용자 → 카카오에 로그인
카카오 → 서비스에 "이 사람 인증됐어요" 토큰 전달
서비스 → 토큰으로 사용자 식별 (비밀번호는 서비스가 모른다)
```


---


## SSO vs OAuth 차이 정리


헷갈리기 쉬운 포인트만 짚는다.

- **SSO** = "나 한 번만 로그인할게" → 여러 시스템에 자동 접속
- **OAuth** = "내 비밀번호는 안 줄게" → 대신 권한(토큰)을 위임

SSO는 **사용 편의성(한 번 로그인)** 문제를 해결하고,
OAuth는 **보안(비밀번호 미노출)** 문제를 해결한다.
둘은 목적이 다르며, 함께 쓰이는 경우도 많다.


---


## 시험용 한 줄 정리

> SSO는 **통합 로그인**(한 번 로그인으로 여러 시스템), OAuth는 **권한 위임**(비밀번호 대신 토큰으로 인증)이다.
