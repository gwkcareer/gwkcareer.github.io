## 용어부터 이해하기


| 용어        | 비유          | 예시             |
| --------- | ----------- | -------------- |
| IP 주소     | 집 주소        | 192.168.35.10  |
| 서브넷 마스크   | 동네를 나누는 기준  | 255.255.252.0  |
| 네트워크 주소   | 동네 이름       | 192.168.32.0   |
| 브로드캐스트 주소 | 동네 전체 방송 주소 | 192.168.35.255 |

- **IP 주소** — 네트워크 상에서 내 PC를 식별하는 주소
- **서브넷 마스크** — "어디까지를 같은 네트워크(동네)로 볼 것인가"를 결정
- **네트워크 주소** — 해당 네트워크 자체를 나타내는 주소
- **브로드캐스트 주소** — 해당 네트워크에 속한 모든 장비에게 데이터를 전송할 때 사용하는 주소

## 옥텟(Octet)이란?


IP 주소를 점(`.`)으로 구분한 각 영역을 **옥텟**이라고 한다.


```plain text
192(1옥텟) . 168(2옥텟) . 35(3옥텟) . 10(4옥텟)
```


## 핵심 규칙


서브넷 마스크에서 **왼쪽부터 처음으로 255가 아닌 숫자가 있는 옥텟**에서 계산한다.

- 예시 1: `255.255.252.0` → 3옥텟에서 계산
- 예시 2: `255.255.255.192` → 4옥텟에서 계산

## 범위 구하는 방법


예제 조건은 아래와 같다.


```plain text
IP   : 192.168.35.10
MASK : 255.255.252.0
```


### Step 1. 255가 아닌 값 찾기


마스크에서 처음으로 255가 아닌 값 → `252` (3옥텟)


### Step 2. 256에서 빼기


```plain text
256 - 252 = 4
```


### Step 3. 해당 옥텟을 4 단위로 나누기


3옥텟 기준으로 범위를 나열하면 다음과 같다.


```plain text
0  ~ 3
4  ~ 7
8  ~ 11
...
32 ~ 35   ← 여기!
36 ~ 39
...
```


### Step 4. 내 IP가 속한 구간 찾기


3옥텟이 `35` → `32 ~ 35` 구간에 속한다.


### Step 5. 네트워크 주소 (구간의 시작)


```plain text
192.168.32.0
```


### Step 6. 브로드캐스트 주소 (구간의 끝)


```plain text
192.168.35.255
```


## 시험에서 묻는 것은 결국 같다


| 문제 유형          | 풀이                          |
| -------------- | --------------------------- |
| 네트워크 주소는?      | 구간의 시작 주소                   |
| 브로드캐스트 주소는?    | 구간의 마지막 주소                  |
| 같은 네트워크인가?     | 두 IP가 같은 구간에 속하는지 확인        |
| 사용 가능한 IP 범위는? | 네트워크 주소 + 1 ~ 브로드캐스트 주소 - 1 |


## AND 연산은 꼭 해야 할까?


정석은 `IP 주소 AND 서브넷 마스크`지만, 시험에서는 아래 3단계가 훨씬 빠르다.

1. 255가 아닌 숫자 찾기
2. 256에서 빼기
3. 범위 찾기

AND 연산은 개념 이해용으로 알아두되, 실전에서는 이 흐름대로 푸는 것이 더 빠르고 실수도 적다.


## 한 줄 요약

> 서브넷 마스크에서 처음으로 255가 아닌 숫자를 찾고, 256 - 해당 값으로 범위 단위를 계산한 뒤, 내 IP가 속한 구간을 찾는다. 구간의 시작 = 네트워크 주소, 구간의 끝 = 브로드캐스트 주소.
